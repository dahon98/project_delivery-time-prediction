# Прогнозирование времени доставки еды 

Данный проект посвящен исследованию набора данных о доставке еды и построению моделей машинного обучения для точного прогнозирования времени доставки заказа. В ходе работы проводится всесторонний разведочный анализ данных (EDA), выявляются ключевые факторы, влияющие на скорость доставки, и выполняется сравнение различных алгоритмов регрессии.

## Описание задачи

Основная цель — построить модель, которая по характеристикам заказа, курьера и внешним условиям сможет предсказать время доставки в минутах. В качестве метрик качества используются **MAE**, **R2** и **RMSE**.

Данные для анализа взяты с Kaggle: [Food Delivery Dataset](https://www.kaggle.com/datasets/gauravmalik26/food-delivery-dataset/data/code).

## Основные этапы работы

### 1. Загрузка и первичный анализ данных
*   Загрузка данных из `train.csv`.
*   Первичный осмотр структуры данных (`df.info()`, `df.describe()`), переименование столбцов для удобства работы.

### 2. Предобработка и очистка данных
*   **Обработка пропусков:** 
*   **Работа с датой и временем:**
    *   Обработка даты и времени заказа и его получения курьером.
    *   Корректный расчет времени, затраченного на подготовку заказа (`time_to_pick_min`), с учетом перехода через полночь.
    *   Очистка целевой переменной `Time_taken(min)` от текстового формата `(min) 24`.
*   **Преобразование типов данных:** Приведение колонок к корректным типам (`datetime`, `timedelta`, `category`, `int`).

### 3. Разведочный анализ данных (EDA)

#### Анализ влияния категориальных признаков
*   **Город (`city`):** Самая быстрая доставка в `urban` (медиана ~22 мин), самая медленная и вариативная — в `semi-urban` (~48 мин).
*   **Погодные условия (`weather_conditions`):** Ясная погода (`sunny`) — самая быстрая (~20 мин). Плохая погода (`cloudy`, `fog`) увеличивает время доставки до ~30-32 мин.
*   **Плотность трафика (`road_traffic_density`):** Прямая зависимость: чем выше загруженность, тем дольше доставка. Пробки (`jam`) приводят к наибольшему времени и разбросу.
*   **Наличие праздника (`festival`):** Очень сильный фактор. В праздничные дни время доставки возрастает почти вдвое (с ~25 до ~45 мин).
*   **Тип заказа и тип транспорта:** Не выявлено существенного влияния на время доставки.

#### Анализ числовых признаков
*   Был создан новый признак — расстояние доставки (`dist_km`), рассчитанное по координатам ресторана и точки доставки.
*   **Распределения:**
    *   `delivery_person_age`: распределение близко к равномерному (20-40 лет).
    *   `delivery_person_ratings`: большинство рейтингов сосредоточено в диапазоне 4.3-5.0.
    *   `dist_km`: распределение сильно смещено вправо. Обнаружены явные выбросы (>50 км), которые были удалены, а затем признак был логарифмирован для нормализации.
    *   `order_hour`: Заказы распределены биполярно, с пиками утром (9-12) и вечером (18-22).

#### Анализ таргета (`time_taken_min_clean`)
*   Распределение целевой переменной имеет правосторонний перекос. Основная масса доставок занимает от 15 до 35 минут, однако есть "хвост" до 50+ минут, что указывает на наличие задержек.
*   Медианное время доставки находится чуть выше 25 минут.

### 4. Проектирование признаков (Feature Engineering)
*   **Логарифмирование расстояния:** Создан признак `dist_km_log = log1p(dist_km)` для борьбы со скошенностью распределения.
*   **Циклическое кодирование времени:** Признак `order_hour` преобразован в две гармоники: `hour_sin` и `hour_cos`, чтобы учесть циклическую природу времени суток.
*   **Кодирование категориальных признаков:**
    *   Признак `festival` преобразован в бинарный (0/1).
    *   Для остальных категориальных признаков (`city`, `weather_conditions`, `road_traffic_density`, `type_of_order`, `type_of_vehicle`, `multiple_deliveries`) применен метод **One-Hot Encoding**.
    *   (В коде также сохранена версия `df_cat` для обучения моделей, поддерживающих категориальные признаки, например, CatBoost).

### 5. Подготовка к обучению
*   Данные разделены на обучающую (`X_train`, `y_train`) и тестовую (`X_test`, `y_test`) выборки в соотношении 80/20.
*   Выполнено масштабирование признаков с помощью `StandardScaler` для корректной работы линейных моделей.

## Результаты

*   **Модель 1 (LinearRegression):** `MAE: 4.74, RMSE: 5.97, R2: 0.61`
*   **Модель 2 (RandomForestRegressor):** `MAE: 3.05, RMSE: 3.79, R2: 0.84`
*   **Модель 3 (CatBoostRegressor):** `MAE: 2.9, RMSE: 3.69, R2: 0.84`

Наилучший результат показала модель **CatBoostRegressor**, которая лучше других справилась с учетом категориальных зависимостей.
